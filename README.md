## ฟังก์ชันหลัก

- เลือกประเภทการเดินทางระหว่างรถไฟไทยและ Airport Rail Link
- ตาราง Airport Rail Link แยกวันจันทร์–ศุกร์ และวันเสาร์–อาทิตย์/วันหยุด โดยใช้ตารางภาพชุดล่าสุด
- เปลี่ยนตัวเลือกสถานี ผลการค้นหา และตารางเต็มตามประเภทการเดินทางอัตโนมัติ
- เลือกตารางปกติหรือตารางพิเศษตามวันที่
- เลือกต้นทาง ปลายทาง วันที่ และเวลาเดินทาง
- แสดง 3 ขบวนที่เหมาะกับเวลาที่เลือกก่อน
- แสดงเวลาตามตาราง เวลาเดินทาง และเวลานับถอยหลัง
- รองรับสถานะรวมและสถานะรายขบวนจาก `data/status.json`
- แยกเวลาตามตารางออกจากเวลาประมาณการเมื่อมีรายงานความล่าช้า
- เตือนเมื่อข้อมูลสถานะหมดอายุ
- บันทึกเส้นทางโปรดและเส้นทางล่าสุดในอุปกรณ์
- แชร์ลิงก์ที่จำค่าต้นทาง ปลายทาง วันที่ และเวลา
- แชร์ตำแหน่งปัจจุบันเป็นลิงก์ Google Maps หลังผู้ใช้อนุญาต Location
- Sync ช่องออกเดินทางกับเวลาปัจจุบัน และมีปุ่มใช้เวลานี้
- ล้างข้อมูลเส้นทางโปรดและรายการล่าสุดได้จากหน้า Planner
- ตั้งแจ้งเตือนก่อนรถออก 10 นาทีขณะเปิดแอพอยู่
- ติดตั้งเป็น PWA และเปิดข้อมูลที่เคยโหลดแบบ Offline
- ตรวจสอบโครงสร้างตารางอัตโนมัติก่อนเริ่มแอพ
- ใช้เวลาเขต `Asia/Bangkok` โดยไม่ขึ้นกับ Timezone ของอุปกรณ์
- แสดงจุดเชื่อมต่อ BTS, MRT, เรือคลองแสนแสบ, สายสีเหลือง และสนามบิน โดยไม่ใช้โลโก้หน่วยงาน
- แสดงดัชนีคุณภาพอากาศ US AQI พร้อมค่า PM2.5 และ PM10 ของกรุงเทพมหานคร

> ตัวเลือก Airport Rail Link มีเฉพาะพญาไทและสุวรรณภูมิ พร้อมเวลาออกทั้งสองทิศทางตามภาพชุดล่าสุด
> เวลาถึงปลายทางคำนวณจากเวลาออกบวกค่าคงที่ `ARL_RUN_MINUTES` ซึ่งตั้งไว้ที่ 50 นาที แก้ได้ที่จุดเดียวใน `index.html`

## ติดตั้งบน GitHub Pages

1. นำไฟล์ทั้งหมดในโฟลเดอร์ `paiduyna` วางที่รากของ Repository
2. Commit และ Push
3. เข้า Settings > Pages
4. เลือก Branch `main` และโฟลเดอร์ `/ (root)`
5. รอประมาณ 1 ถึง 3 นาทีแล้วเปิดเว็บไซต์

```bash
git add .
git commit -m "PAI.DUY.NA Full build 2026-07-22 v4"
git push
```

หลังอัปเดต Service Worker อาจเก็บหน้าเก่าไว้ชั่วคราว ให้กดรีเฟรชหรือปิดแล้วเปิดแอพอีกครั้ง

## ทดสอบในเครื่อง

ห้ามดับเบิลคลิก `index.html` เพราะเบราว์เซอร์อาจบล็อก JSON และ Service Worker ให้รัน Local Server

```bash
cd paiduyna
python3 -m http.server 8000
```

จากนั้นเปิด `http://localhost:8000`

## เผยแพร่ข่าวจาก iPhone หรือคอมพิวเตอร์

เปิด `https://ชื่อเว็บไซต์ของคุณ/admin/` แล้วตั้งค่าครั้งแรก:

1. กรอกเจ้าของ Repository และชื่อ Repository
2. Branch ใช้ `main`
3. ตำแหน่งไฟล์ใช้ `data/status.json`
4. วาง Fine-grained Personal Access Token ที่เลือกเฉพาะ Repository นี้ และมีสิทธิ์ Contents: Read and write
5. กด `ทดสอบการเชื่อมต่อ`

การใช้งานครั้งต่อไป:

1. เลือกระดับข่าว ปกติ ล่าช้า หรือฉุกเฉิน
2. กรอกข้อความและกำหนดเวลาหมดอายุ
3. กด `แสดงตัวอย่าง`
4. กด `เผยแพร่ทันที`

หน้า Admin ออนไลน์แก้เฉพาะข่าวใน `data/status.json` ก่อนเผยแพร่ระบบจะอ่านไฟล์เดิมและรักษาฟิลด์อื่นไว้ ไม่แก้ `index.html` ตารางเวลา เส้นทาง สถานี หรือระบบค้นหา
