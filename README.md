## ติดตั้งบน GitHub Pages

1. นำไฟล์ทั้งหมดในโฟลเดอร์ `paiduyna` วางที่รากของ Repository
2. Commit และ Push
3. เข้า Settings > Pages
4. เลือก Branch `main` และโฟลเดอร์ `/ (root)`
5. รอประมาณ 1 ถึง 3 นาทีแล้วเปิดเว็บไซต์

```bash
git add .
git commit -m "PAI.DUY.NA Full build 2026-07-22 v4"
git push
```

หลังอัปเดต Service Worker อาจเก็บหน้าเก่าไว้ชั่วคราว ให้กดรีเฟรชหรือปิดแล้วเปิดแอพอีกครั้ง

## ทดสอบในเครื่อง

ห้ามดับเบิลคลิก `index.html` เพราะเบราว์เซอร์อาจบล็อก JSON และ Service Worker ให้รัน Local Server

```bash
cd paiduyna
python3 -m http.server 8000
```

จากนั้นเปิด `http://localhost:8000`

## อัปเดตสถานะด้วย Mobile Admin

หน้าเว็บหลักไม่มีลิงก์ Admin แต่ผู้ดูแลเปิดหน้า `admin.html` โดยตรงจาก Safari บน iPhone ได้

1. เปิด `https://somchais1113-ui.github.io/Paiduyna2/admin.html`
2. กรอก Repository owner เป็น `somchais1113-ui`, Repository เป็น `Paiduyna2`, Branch เป็น `main` และ Path เป็น `data/status.json`
3. วาง Fine-grained Token ที่เลือกเฉพาะ Repo นี้และสิทธิ์ Contents: Read and write
4. หากเป็น iPhone ส่วนตัว ให้เลือก `จำ Token` แล้วกด `ทดสอบการเชื่อมต่อ`
5. เลือกระดับสถานะ ปกติ ล่าช้า หรือฉุกเฉิน
6. กรอกข้อความและกำหนดเวลาหมดอายุ
7. หากต้องการแจ้งรายขบวน ให้กรอกหนึ่งบรรทัดต่อหนึ่งขบวน

```text
379 | warn | 15 | ล่าช้าประมาณ 15 นาที
1213 | alert | 0 | โปรดตรวจสอบกับเจ้าหน้าที่ก่อนเดินทาง
```

รูปแบบคือ:

```text
เลขขบวน | ok/warn/alert | ล่าช้านาที | ข้อความ
```

8. กด `แสดงตัวอย่าง` เพื่อตรวจสอบข้อมูล
9. กด `เผยแพร่ทันที` แล้วรอ GitHub Pages Build ประมาณ 1 ถึง 2 นาที

หน้า Admin เป็น URL สาธารณะและไม่มีระบบล็อกอิน ผู้ที่ไม่มี Token จะเขียนข้อมูลไม่ได้ ความปลอดภัยจึงขึ้นอยู่กับ Token โดยตรง ห้ามฝัง Token ในโค้ด ห้ามส่ง Token ทางแชตหรือภาพหน้าจอ และหากโทรศัพท์หายให้ Revoke Token จาก GitHub ทันที
