## 1. โครงสร้างไฟล์

```
paiduyna/
├─ index.html               ตัวเว็บทั้งหมด แก้เมื่อเปลี่ยนตารางเดินรถหรือดีไซน์
├─ .nojekyll                บอก GitHub Pages ให้เสิร์ฟไฟล์ตรง ๆ ไม่ผ่าน Jekyll
├─ README.md                ไฟล์นี้
└─ data/
   ├─ status.json           ข่าวสถานะการเดินรถ แก้บ่อยที่สุด
   └─ alternatives.json     ทางเลือกสำรองในการเดินทาง
```

**ห้ามย้าย `data/` ออกจากโฟลเดอร์เดียวกับ `index.html`** หากจำเป็นต้องย้ายจริง
ให้แก้ 2 บรรทัดบนสุดของแท็ก `<script>` ใน `index.html`

```js
const STATUS_URL = "data/status.json";
const ALT_URL    = "data/alternatives.json";
```

## 2. การติดตั้งบน GitHub Pages

1. แตกไฟล์ zip แล้วนำไฟล์ทั้งหมดวางที่ราก (root) ของ repository
2. สั่ง commit และ push

```bash
git add index.html .nojekyll README.md data/
git commit -m "PAI.DUY.NA build 2026-07-22c"
git push
```

3. เข้า Settings ของ repository ไปที่หัวข้อ Pages เลือก Branch เป็น `main` และโฟลเดอร์เป็น `/ (root)`
4. รอประมาณ 1 ถึง 3 นาที แล้วเปิดเว็บ เลื่อนลงล่างสุดเพื่อตรวจสอบว่าขึ้นข้อความ
   `PAI.DUY.NA build 2026-07-22c` หากยังไม่ขึ้น แปลว่าเซิร์ฟเวอร์ยังเสิร์ฟไฟล์เก่าอยู่

## 3. การทดสอบในเครื่องก่อนเผยแพร่

ห้ามเปิดด้วยการดับเบิลคลิกไฟล์ เพราะเบราว์เซอร์จะบล็อกการอ่านไฟล์ JSON
ตามนโยบายความปลอดภัย CORS ให้รันเซิร์ฟเวอร์ในเครื่องแทน

```bash
cd paiduyna
python3 -m http.server 8000
```

จากนั้นเปิด `http://localhost:8000`

## 4. การอัปเดตข่าวสถานะ

แก้ไขไฟล์ `data/status.json` ไฟล์เดียว ไม่ต้องแตะ `index.html`

```json
{
  "level": "ok",
  "tag": "อัปเดตสถานะการเดินรถ",
  "updatedAt": "22/07/2569 18:30 น.",
  "message": "ข้อความหลักที่ต้องการประกาศ",
  "items": ["รายละเอียดข้อที่หนึ่ง", "รายละเอียดข้อที่สอง"]
}
```

ค่า `level` กำหนดสีของแถบประกาศ

| ค่า | สี | ใช้เมื่อ |
| --- | --- | --- |
| `ok` | เขียว | เดินรถปกติ |
| `warn` | เหลือง | ล่าช้าหรือมีข้อควรระวัง |
| `alert` | แดง | งดเดินรถหรือเหตุฉุกเฉิน |

หน้าเว็บดึงข้อมูลใหม่อัตโนมัติทุก 10 นาที และผู้ใช้กดปุ่มรีเฟรชข้อมูลเองได้ตลอดเวลา

## 5. การอัปเดตทางเลือกสำรอง

แก้ไขไฟล์ `data/alternatives.json`

```json
{
  "updatedAt": "22/07/2569",
  "items": [
    {
      "title": "ชื่อทางเลือก",
      "detail": "คำอธิบายสั้น ๆ",
      "note": "ข้อควรทราบเพิ่มเติม ใส่หรือไม่ใส่ก็ได้"
    }
  ]
}
```

ข้อมูลชุดที่แถมมาเป็นเพียงโครงตั้งต้น **ต้องกรอกข้อมูลจริงจากประกาศของผู้ให้บริการ
ก่อนเผยแพร่ให้ผู้โดยสารใช้งาน** ตราบใดที่ยังไม่แก้ กล่องเตือนสีเหลืองจะยังแสดงอยู่

## 6. การแก้ไขตารางเดินรถ

ข้อมูลอยู่ใน `index.html` แบ่งเป็น 4 ชุด

| ตัวแปร | เนื้อหา | จำนวนช่องเวลาต่อขบวน |
| --- | --- | --- |
| `DAILY_UP` | ตารางปกติ เที่ยวขึ้น | 21 |
| `DAILY_DOWN` | ตารางปกติ เที่ยวล่อง | 21 |
| `SPECIAL_UP` | ตารางพิเศษ ไปลาดกระบัง | 8 |
| `SPECIAL_DOWN` | ตารางพิเศษ ไปพญาไท | 8 |

**กฎสำคัญ** จำนวนช่องเวลาในแต่ละขบวนต้องเท่ากับจำนวนสถานีของตารางนั้นเสมอ
สถานีที่ขบวนไม่จอดให้ใส่ `null` ห้ามข้ามหรือเว้นว่าง มิฉะนั้นคอลัมน์จะเหลื่อมกันทั้งตาราง

ลำดับเวลาของเที่ยวล่องเรียงกลับด้านกับเที่ยวขึ้น คือเริ่มจากสถานีปลายทางของสาย

## 7. เมื่อหน้าเว็บทำงานผิดปกติ

หน้าเว็บมีระบบแจ้งข้อผิดพลาดในตัว หากสคริปต์ทำงานผิดพลาด
จะขึ้นแถบสีแดงพร้อมข้อความและหมายเลขบรรทัดที่ผิดให้เห็นบนหน้าจอทันที
โดยไม่ต้องเปิดเครื่องมือนักพัฒนา ซึ่งบนโทรศัพท์มือถือทำได้ยาก

หากช่องเลือกสถานีว่างเปล่าและนาฬิกาค้างที่ `--:--:--` แปลว่าสคริปต์ไม่ทำงานตั้งแต่ต้น
ให้ตรวจสอบตามลำดับนี้

1. ดูรหัสเวอร์ชันท้ายหน้าว่าตรงกับไฟล์ที่เพิ่งอัปโหลดหรือไม่
2. เปิดหน้าเว็บในโหมดส่วนตัวเพื่อตัดปัญหาแคชของเบราว์เซอร์
3. รอ 2 ถึง 5 นาทีเพื่อให้แคชของ GitHub Pages หมดอายุ
4. ตรวจสอบว่า commit เข้า branch ที่ตั้งค่าไว้ในหน้า Pages จริง
