## ฟังก์ชันหลัก

- เลือกตารางปกติหรือตารางพิเศษตามวันที่
- เลือกต้นทาง ปลายทาง วันที่ และเวลาเดินทาง
- แสดง 3 ขบวนที่เหมาะกับเวลาที่เลือกก่อน
- แสดงเวลาตามตาราง เวลาเดินทาง และเวลานับถอยหลัง
- รองรับสถานะรวมและสถานะรายขบวนจาก `data/status.json`
- แยกเวลาตามตารางออกจากเวลาประมาณการเมื่อมีรายงานความล่าช้า
- เตือนเมื่อข้อมูลสถานะหมดอายุ
- บันทึกเส้นทางโปรดและเส้นทางล่าสุดในอุปกรณ์
- แชร์ลิงก์ที่จำค่าต้นทาง ปลายทาง วันที่ และเวลา
- แชร์ตำแหน่งปัจจุบันเป็นลิงก์ Google Maps หลังผู้ใช้อนุญาต Location
- Sync ช่องออกเดินทางกับเวลาปัจจุบัน และมีปุ่มใช้เวลาตอนนี้
- ล้างข้อมูลเส้นทางโปรดและรายการล่าสุดได้จากหน้า Planner
- ตั้งแจ้งเตือนก่อนรถออก 10 นาทีขณะเปิดแอพอยู่
- ติดตั้งเป็น PWA และเปิดข้อมูลที่เคยโหลดแบบ Offline
- ตรวจสอบโครงสร้างตารางอัตโนมัติก่อนเริ่มแอพ
- ใช้เวลาเขต `Asia/Bangkok` โดยไม่ขึ้นกับ Timezone ของอุปกรณ์

## ติดตั้งบน GitHub Pages

1. นำไฟล์ทั้งหมดในโฟลเดอร์ `paiduyna` วางที่รากของ Repository
2. Commit และ Push
3. เข้า Settings > Pages
4. เลือก Branch `main` และโฟลเดอร์ `/ (root)`
5. รอประมาณ 1 ถึง 3 นาทีแล้วเปิดเว็บไซต์

```bash
git add .
git commit -m "PAI.DUY.NA Full build 2026-07-22 v2"
git push
```

หลังอัปเดต Service Worker อาจเก็บหน้าเก่าไว้ชั่วคราว ให้กดรีเฟรชหรือปิดแล้วเปิดแอพอีกครั้ง

## ทดสอบในเครื่อง

ห้ามดับเบิลคลิก `index.html` เพราะเบราว์เซอร์อาจบล็อก JSON และ Service Worker ให้รัน Local Server

```bash
cd paiduyna
python3 -m http.server 8000
```

จากนั้นเปิด `http://localhost:8000`

## อัปเดตสถานะด้วยเครื่องมือ Local Admin

หน้าเว็บสาธารณะไม่มีปุ่ม Admin เพื่อไม่ให้ผู้ใช้งานทั่วไปเข้าถึงเครื่องมืออัปเดตข่าว

1. เปิดไฟล์ `../admin-local/PAI_DUY_NA_Admin.html` จากแพ็กเกจต้นฉบับ
2. เลือกระดับสถานะ ปกติ ล่าช้า หรือฉุกเฉิน
3. กรอกข้อความและกำหนดเวลาหมดอายุ
4. หากต้องการแจ้งรายขบวน ให้กรอกหนึ่งบรรทัดต่อหนึ่งขบวน

```text
379 | warn | 15 | ล่าช้าประมาณ 15 นาที
1213 | alert | 0 | โปรดตรวจสอบกับเจ้าหน้าที่ก่อนเดินทาง
```

รูปแบบคือ:

```text
เลขขบวน | ok/warn/alert | ล่าช้านาที | ข้อความ
```

5. กด `แสดงตัวอย่าง` เพื่อตรวจสอบข้อมูล
6. กด `Export status.json`
7. นำไฟล์ที่ดาวน์โหลดไปแทน `data/status.json` ใน GitHub

อย่าอัปโหลดโฟลเดอร์ `admin-local` ขึ้น GitHub Pages สิทธิ์แก้ข้อมูลจริงควรให้เฉพาะบัญชี GitHub ของผู้ดูแล Repository ที่มีสิทธิ์ Write เท่านั้น

## โครงสร้าง status.json

```json
{
  "mode": "manual",
  "verified": true,
  "level": "warn",
  "tag": "อัปเดตสถานะการเดินรถ",
  "updatedAt": "22/07/2569 18:30 น.",
  "updatedAtISO": "2026-07-22T18:30:00+07:00",
  "expiresAtISO": "2026-07-22T23:59:00+07:00",
  "source": "ผู้ดูแล PAI.DUY.NA",
  "message": "มีขบวนล่าช้าบางส่วน",
  "items": ["โปรดเผื่อเวลาเดินทาง"],
  "trains": [
    {
      "no": "379",
      "level": "warn",
      "delayMinutes": 15,
      "message": "ล่าช้าประมาณ 15 นาที"
    }
  ]
}
```

`mode: manual` หมายถึงข้อมูลที่ผู้ดูแลกรอก ไม่ใช่ตำแหน่งรถไฟจากระบบ GPS หรือ API ของผู้ให้บริการ

## ทางเลือกสำรอง

แก้ `data/alternatives.json` และเปลี่ยน `verified` เป็น `true` เมื่อข้อมูลผ่านการตรวจสอบแล้ว

```json
{
  "verified": true,
  "updatedAt": "22/07/2569",
  "source": "ประกาศของผู้ให้บริการ",
  "items": []
}
```

หาก `verified` ไม่ใช่ `true` หน้าเว็บจะแสดงกล่องเตือนโดยอัตโนมัติ

## ข้อจำกัดของ Full Version แบบ GitHub Pages

- ไม่สามารถติดตามตำแหน่งรถจริงได้หากไม่มี API หรือ Feed จากผู้ให้บริการ
- ต้อง Export และอัปโหลด `status.json` เพื่อให้ผู้ใช้ทุกคนเห็นข้อมูลเดียวกัน
- การแจ้งเตือนแบบปิดแอพแล้วยังทำงาน ต้องมี Push Server เพิ่มเติม
- การแชร์ตำแหน่งจะทำงานเมื่อเว็บไซต์ใช้ HTTPS และผู้ใช้อนุญาต Location เท่านั้น
